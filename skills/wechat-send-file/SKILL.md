# 微信文件发送自动化技能 (Python版本)

这个技能使用Python自动化控制微信桌面客户端，通过**模板匹配**技术定位微信界面元素，自动化完成搜索用户和发送文件的完整流程。基于wechat-call-python的成熟架构，使用纯Python实现。

## 重要特性

1. **文件路径识别** - 智能解析用户消息中的文件路径
2. **图形匹配技术** - 使用模板匹配定位搜索框和上传文件按钮
3. **三次回车操作** - 通过三次回车键快速进入对话框
4. **跨平台支持** - 支持Windows、macOS和Linux
5. **智能重试机制** - 支持多次尝试扫描，提高成功率
6. **模拟人类操作** - 添加随机偏移模拟自然点击
7. **自动提取信息** - 从消息中自动提取用户名和文件路径

## 触发条件

当用户消息匹配以下模式时触发：
- "把[文件位置]的文件用微信给[用户名]"
- "用微信给[用户名]发送[文件位置]的文件"
- "给[用户名]发送[文件位置]的微信文件"
- "通过微信把[文件位置]的文件发给[用户名]"
- "微信发送[文件位置]的文件给[用户名]"

**示例**:
- "把C:\Users\Administrator\Documents\report.docx的文件用微信给可乐煮鸡"
- "用微信给张三发送D:\work\project.pdf的文件"
- "把桌面\照片.jpg的文件用微信发给李四"

## 前提条件

1. **微信桌面客户端** 必须已安装并登录
2. **Python环境** (Python 3.8+)
3. **必要Python包**：
   - pyautogui: 鼠标键盘控制
   - pillow (PIL): 图像处理
   - pyperclip: 剪贴板操作
   - opencv-python: 模板匹配 (可选，提供更好的匹配)
4. **模板图片** `soupic.png` 和 `upload_file.png` 需正确准备
5. **文件存在性**：指定的文件路径必须存在且可访问

### 2. 发送文件
```bash
# 使用Python脚本
python wechat_send_file.py --message "把C:\Users\Administrator\Documents\report.docx的文件用微信给可乐煮鸡"

# 带调试信息
python wechat_send_file.py --message "把test.pdf的文件用微信给测试用户" --debug

# 自定义置信度阈值
python wechat_send_file.py --message "把photo.jpg的文件用微信给朋友" --confidence 0.85 --debug
```

## 配置参数

### 命令行参数
- `--message`: **必需**，完整消息内容
- `--debug`: 可选，显示详细调试信息
- `--simulate`: 可选，模拟模式，不执行实际操作
- `--confidence <数值>`: 可选，模板匹配置信度阈值 (0.0-1.0, 默认0.85)
- `--use-opencv`: 可选，使用OpenCV进行模板匹配（如果安装了OpenCV）
- `--no-opencv`: 可选，不使用OpenCV（即使可用）
- `--upload-confidence <数值>`: 可选，上传文件按钮匹配置信度阈值 (0.0-1.0, 默认0.9)

### 匹配参数
- 最大扫描尝试次数：5次
- 扫描间隔延迟：500ms
- 扫描步进：2像素（提高效率）
- 随机偏移范围：±5像素
- 点击延迟：100ms

## 技术实现

### 核心模块
1. **消息解析器** - 智能提取用户名和文件路径
2. **文件验证器** - 验证文件存在性和可访问性
3. **模板匹配器** - 基于wechat-call-python的成熟匹配算法
4. **上传文件操作器** - 专门处理文件上传对话框
5. **Windows对话框控制器** - 处理Windows文件选择对话框

### 工作流程

#### 1. 消息解析和文件验证
- 从用户消息中提取用户名和文件路径
- 使用正则表达式智能匹配各种消息格式
- 验证文件路径是否存在且可读
- 复制用户名到剪贴板，准备搜索

#### 2. 激活微信窗口
- 按 `Win+D` 返回桌面
- 按 `Ctrl+Alt+W` 唤醒微信窗口
- 等待窗口完全加载

#### 3. 模板匹配查找搜索图标
- 截取当前屏幕图像
- 使用`soupic.png`模板进行匹配
- 智能多次扫描，直到找到匹配位置
- 在匹配位置的中心点击（添加随机偏移）

#### 4. 搜索用户并进入对话框
- 粘贴用户名到搜索框
- 按回车键确认搜索
- **连续按两次回车键**快速进入对话框（总计三次回车）
- 等待对话框完全加载

#### 5. 模板匹配查找上传文件按钮
- 截取当前屏幕图像
- 使用`upload_file.png`模板进行匹配
- 智能多次扫描，直到找到匹配位置
- 在匹配位置的中心点击（添加随机偏移）

#### 6. 选择文件并发送
- 激活Windows文件选择对话框
- 粘贴文件路径到对话框
- 按回车键确认选择
- 按回车键发送文件（部分版本可能需要额外回车）

#### 7. 自动清理桌面截图
- 文件发送完成后
- 自动搜索桌面上的截图文件
- 删除所有匹配模式的截图文件
- 保持桌面整洁

### 关键设计

#### 1. 三次回车策略
```python
# 第一次回车：确认搜索用户
pyautogui.press('enter')
time.sleep(0.3)

# 第二次回车：选择用户
pyautogui.press('enter')
time.sleep(0.3)

# 第三次回车：进入对话框（适用于部分微信版本）
pyautogui.press('enter')
time.sleep(0.5)
```

#### 2. 文件路径处理
```python
def validate_file_path(file_path):
    """验证文件路径是否有效"""
    # 标准化路径
    path = os.path.expandvars(os.path.expanduser(file_path))
    
    # 验证存在性和可访问性
    if not os.path.exists(path):
        raise FileNotFoundError(f"文件不存在: {path}")
    
    if not os.path.isfile(path):
        raise ValueError(f"路径不是文件: {path}")
    
    return os.path.abspath(path)
```

#### 3. Windows文件对话框操作
```python
def select_file_in_dialog(file_path):
    """在Windows文件选择对话框中选择文件"""
    # 等待对话框出现（使用快捷键Alt+N聚焦到文件名输入框）
    time.sleep(0.5)
    pyautogui.hotkey('alt', 'n')
    time.sleep(0.2)
    
    # 粘贴完整文件路径
    pyperclip.copy(file_path)
    pyautogui.hotkey('ctrl', 'v')
    time.sleep(0.3)
    
    # 回车确认选择
    pyautogui.press('enter')
```

## 故障排除

### 常见问题

**问题**: 无法找到模板匹配
**解决**: 
1. 降低置信度阈值：`--confidence 0.8 --upload-confidence 0.85`
2. 检查模板图片是否正确
3. 确保微信界面已完全加载
4. 确保截图区域包含目标元素

**问题**: 文件路径解析失败
**解决**: 
1. 检查消息格式是否符合预期模式
2. 确保文件路径用引号包裹（如果包含空格）
3. 检查文件路径是否有效且存在

**问题**: Windows文件对话框操作失败
**解决**: 
1. 确保文件对话框已正确弹出
2. 尝试手动操作一次以了解对话框行为
3. 检查是否有特殊字符导致路径无效

**问题**: 三次回车未进入对话框
**解决**: 
1. 根据微信版本调整回车次数
2. 添加额外延迟等待对话框加载
3. 检查是否有其他用户同名需要选择

**问题**: 依赖安装失败
**解决**: 
1. 确保Python版本 >= 3.8
2. 使用国内镜像：`pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple`
3. 确保网络连接正常
