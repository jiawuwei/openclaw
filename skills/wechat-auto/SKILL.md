# 微信消息自动化技能 (Python v3.0版)

这个技能是**基于wechat-call-python技能重构**的微信消息自动化工具。完全使用wechat-call-python的模板匹配技术，专注于消息发送功能。使用Python实现，通过**模板匹配**技术定位微信搜索框(soupic.png)和输入框(input.png)，自动化完成搜索用户和发送消息的完整流程。

## 版本特点

1. **基于wechat-call-python的成熟架构** - 使用完全相同的代码结构和技术栈
2. **专注消息发送** - 专为发送消息优化，流程更简洁
3. **模板匹配技术** - 使用soupic.png匹配搜索框，input.png匹配输入框
4. **智能消息解析** - 支持多种消息格式，自动提取用户名和消息内容
5. **完善的错误处理** - 包含详细的日志和重试机制
6. **自动清理截图** - 完成后自动删除所有桌面截图文件

## 触发条件

当用户消息匹配以下模式时触发：
- "用微信给[用户名]发[消息内容]"
- "用微信给[用户名]发送[消息内容]"
- "发送[消息内容]给[用户名]"
- "[用户名] [消息内容]" (简单格式)

## 前提条件

1. **微信桌面客户端** 必须已安装并登录
2. **Python环境** (Python 3.8+)
3. **必要Python包**：
   - pyautogui: 鼠标键盘控制
   - pillow (PIL): 图像处理
   - pyperclip: 剪贴板操作（可选，有回退方案）
4. **模板图片** `soupic.png` 和 `input.png` 需正确准备

## 快速使用

### 1. 直接运行脚本
```bash
# 进入技能目录
cd skills/wechat-auto

# 使用完整格式
python wechat_auto_message.py --user "用微信给可乐煮鸡发今天不吃饭"

# 带调试信息
python wechat_auto_message.py --user "用微信给测试用户发测试消息" --debug

# 模拟模式（不执行实际操作）
python wechat_auto_message.py --user "用微信给测试用户发测试消息" --simulate --debug
```

### 2. 使用批处理文件
```bash
# 运行批处理文件
send_message.bat "用微信给可乐煮鸡发今天不吃饭"
```

### 3. 命令行参数
```bash
# 完整参数列表
python wechat_auto_message.py --user "完整消息" [选项]

# 参数说明
--user        必需，用户名或完整消息（如"用微信给XXX发YYY"）
--message     可选，消息内容（如果在--user中已包含则不需要）
--debug       可选，显示详细调试信息
--simulate    可选，模拟模式，不执行实际操作
--confidence  可选，模板匹配置信度阈值 (0.0-1.0, 默认0.85)
--use-opencv  可选，使用OpenCV进行模板匹配（如果安装了OpenCV）
--no-opencv   可选，不使用OpenCV（即使可用）
```

## 工作流程

### 1. 消息解析
- 从"用微信给[用户名]发[消息内容]"格式中提取用户名和消息
- 分别复制用户名和消息到剪贴板

### 2. 激活微信窗口
- 按 `Win+D` 返回桌面
- 按 `Ctrl+Alt+W` 唤醒微信窗口（假设快捷键可用）
- 等待窗口完全加载

### 3. 模板匹配查找搜索图标
- 截取当前屏幕图像
- 使用`soupic.png`模板进行智能匹配
- 最多5次扫描尝试，提高成功率
- 在匹配位置的中心点击（添加随机偏移）

### 4. 搜索并选择用户
- 粘贴用户名到搜索框
- 按回车键选择用户
- 再次按回车进入对话框

### 5. 模板匹配查找输入框
- 截取当前屏幕图像  
- 使用`input.png`模板进行智能匹配
- 最多5次扫描尝试，提高成功率
- 在匹配位置的中心点击（添加随机偏移）

### 6. 发送消息
- 粘贴消息内容到输入框
- 按回车发送消息

### 7. 清理工作
- 自动搜索桌面上的所有截图文件
- 删除所有匹配模式的截图文件
- 保持桌面整洁

## 技术实现

### 核心代码架构
完全采用wechat-call-python的代码结构：
- **Config类**: 配置参数管理
- **Logger类**: 日志系统
- **Utils类**: 工具函数（剪贴板、延迟、随机偏移等）
- **TemplateMatcher类**: 模板匹配核心（支持PIL和OpenCV）
- **Operations类**: 操作执行封装
- **WeChatSender类**: 主流程控制

### 模板匹配技术
与wechat-call-python使用相同的模板匹配技术：
- **PIL模式**: 使用Pillow库进行灰度匹配
- **OpenCV模式**: 可选，使用OpenCV进行更精确的匹配
- **智能扫描**: 多次尝试，步进扫描提高效率
- **置信度控制**: 可调节的匹配置信度阈值
- **随机偏移**: 添加随机偏移模拟人类操作

## 故障排除

### 常见问题

**问题**: 模板匹配失败
**解决**: 
1. 降低置信度阈值：`--confidence 0.7`
2. 确保模板图片与当前微信版本匹配
3. 确保微信界面已完全加载
4. 检查屏幕分辨率是否合适

**问题**: 剪贴板操作失败
**解决**: 
1. 安装pyperclip：`pip install pyperclip`
2. 确保文本内容不包含特殊字符
3. 脚本会自动使用系统命令作为回退

**问题**: 微信窗口未激活
**解决**: 
1. 确保微信客户端已启动并登录
2. 检查快捷键`Ctrl+Alt+W`是否可用
3. 可以手动激活微信窗口后运行脚本

**问题**: 找不到用户
**解决**: 
1. 确保用户名正确
2. 确保微信中有该用户的对话记录
3. 检查用户名是否包含特殊字符
